# V0.5.0 架构文档

## 概述

V0.5.0 的核心目标是**统一运行时 + 单一 HTTP 服务器 + LangGraph 可选用**。

### 关键变化

| 项目 | V0.4.0 | V0.5.0 |
|------|--------|--------|
| Agent 入口 | `CodexSupervisor` (单一) | `AgentOrchestrator` (统一入口) |
| HTTP 服务器 | 2 个独立 FastAPI 应用 | 1 个统一 FastAPI 应用 |
| 意图处理 | 每个 intent 单独 stub | 全部走 `ReActLoop` |
| LangGraph | 适配器存在但 inert | 通过 `?backend=langgraph` 可选启用 |
| Agent 对话 API | 无 | `/api/v1/agents/conversations/*` + SSE |
| RAG API | 独立服务器 | 挂载到主服务器 `/api/v1/rag/*` |

---

## 统一运行时架构

```
用户消息
  ↓
AgentOrchestrator.run_agent()
  ↓
SupervisorRouter.route()
  ├── _classify_intent() → 意图分类
  └── _run_react_loop()
        ↓
      ReActLoop.run(state)
        ├── LLM think (如已配置)
        ├── ToolContextWrapper.call() ← 注入 AgentState
        └── PluginToolRegistry.call()
  ↓
AgentState (stop_reason, reply, trace)
```

### 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| `AgentOrchestrator` | `agents/runtime/orchestrator.py` | 统一 async/sync 入口 |
| `SupervisorRouter` | `agents/supervisor/router.py` | 意图路由 (5 个 intent) |
| `ReActLoop` | `agents/runtime/loop.py` | Agent 主循环 |
| `ToolContextWrapper` | `agents/runtime/_tool_context.py` | 注入 AgentState 到 tool context |
| `AgentState` | `agents/runtime/state.py` | 对话状态管理 |

## 统一 HTTP 服务器

```
FastAPI app (tools/server/app.py)
  ├── /api/v1/health                  — 健康检查
  ├── /api/v1/pipeline/run            — 流水线执行 (?backend=workflow|langgraph)
  ├── /api/v1/pipeline/stream         — SSE 流式执行
  ├── /api/v1/rag/*                   — RAG 子路由器 (create_rag_router)
  │     ├── /api/v1/rag/query
  │     ├── /api/v1/rag/query/cognitive
  │     ├── /api/v1/rag/documents
  │     └── /ws/v1/query (WebSocket)
  ├── /api/v1/agents/conversations/*  — Agent 对话 API
  │     ├── POST   /                   — 创建对话
  │     ├── GET    /{cid}              — 获取对话状态
  │     ├── DELETE /{cid}              — 删除对话
  │     └── POST   /{cid}/messages     — 发送消息 (SSE)
  └── /api/v1/webhook/*               — Webhook 入口 (V0.4.0 F4)
```

### RAG 子路由器

`create_rag_router(pipeline, observer)` 工厂函数从 `tools/rag/api.py` 提取，
以 `APIRouter(prefix="/api/v1/rag")` 形式挂载到主应用，避免路径冲突。

## LangGraph 集成 (可选)

### 启用方式

```bash
pip install langgraph
curl -X POST http://localhost:8080/api/v1/pipeline/run \
  -d '{"requirement": "构建登录模块", "backend": "langgraph"}'
```

### 调用链

```
PipelineOrchestrator.run_pipeline(backend="langgraph")
  ↓
PipelineCompiler.compile_from_config()
  ↓
CompiledPipeline.to_langgraph(llm_provider, tool_registry)
  ↓
build_pipeline_workflow(compiled) → Workflow
  ↓
LangGraphBackend.build(workflow) → CompiledStateGraph
  ↓
LangGraphBackend.execute(graph, initial_state)
```

### 设计决策

- **Lazy import**: `langgraph` 未安装时，`to_langgraph()` 抛出 `ImportError`，
  PipelineOrchestrator 捕获后返回友好错误消息
- **双后端**: `backend` 参数控制，默认 `"workflow"`，无需 LangGraph 即可运行

## Agent 对话 API

### SSE 事件格式

```
event: intent
data: {"intent": "code_generation"}

event: step
data: {"step": 1, "total": 10}

event: reply
data: {"reply": "收到代码生成请求..."}

event: done
data: {"stop_reason": "answered", "step_count": 0}
```

### 对话生命周期

```
create() → cid
send_message(cid, msg) → SSE stream
get(cid) → AgentState
delete(cid) → bool
```

LRU 淘汰策略: 默认 max 100 条，超出时移除 `access_order` 最小的对话。

---

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 同步兼容 | `asyncio.run()` in `run_agent_sync()` | CLI/旧代码无事件循环 |
| RAG 合并方式 | `APIRouter(prefix="/api/v1/rag")` | 子应用不继承父中间件 |
| 工具 context 注入 | `ToolContextWrapper` 包装器 | 不修改 loop.py 热路径 |
| 对话状态存储 | 内存 dict + LRU(100) | SSE Queue 无法序列化到 SQLite |
| LangGraph 可选 | lazy import + 条件跳过 | 保持零新依赖的核心安装 |
| 旧路径处理 | 保留 `run_full_pipeline()` 委托旧 Phase1/Phase2 | 保护 1446 个已有测试 |
